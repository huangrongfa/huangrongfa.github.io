git学习笔记整理

发表于 2020-02-26 更新于 2020-02-27 分类于 git

前言

git是一个开源的分布式版本控制系统，在现在的前端开发中，是经常使用的版本控制工具，所以，熟练的掌握它的使用，可以提升我们的工作效率，也是我们自身能力的提现，现在整理一下，我在工作中使用的一些基础指令。

git init 这个命令一般是用来初始化git仓库的；
git config – system 该命令是给当前用户一次性设置邮箱和用户名；
git config – local 该命令是给当前项目一次性设置邮箱和用户名；
git config – loacal user.name ‘你的名字’；
git config – loacal user.eamil ‘你的邮箱’；

git add . 该命令可以对修改的文件，进行跟踪；
git commit - m ‘提交注释’ 该命令是对提交信息，做出说明；
git push 该命令推送代码到远程仓库（改变指针）；
git status 查看当前文件的状态；

git pull 此命令用户拉取最新的代码，也就相当于 git fetch + git merge

git branch 查看当前分支；
git branch ‘分支名字’ 该命令可以创建分支；
git checkout 用来切换分支；也可以简写 git branch -b ‘分支名字’；
git branch -d ‘分支名字’ 该命令删除分支（注：自己不能删除自己）,需要切换分支才可以；
git branch -D ‘分支名字’ 表示强制删除分支；
git branch -v 查看分支的信息,通常是sha1值；

git rm ‘.txt’ 删除已经提交的文件（删除之后，文件回到暂存区）&& git commit -m ‘.txt’ 表示彻底删除该文件；(分为2步操作)；

git merge 这个用来合并分支；
git log && git log –graph && git reflog 可以查看提交的日志信息；
git reset –hard HEAD^ 此命令表示回退到某一次commit，^表示一个版本，也可以通过sha1值来进行回退。（例如： git reset –head e82f00）；
git commit –amend 该命令可以用来修改commit提交的信息；

git stash 该命令可以保存现场（未完成的功能，可以执行这个命令），也就是临时保存；
git stash list 查看保存的记录；
git stash pop 释放记录(将原来的保存，只用于还原内容)；

git tag 该命令用来打一个标签(例如： git tag v1.0)；
git tag -d ‘标签名称’ 该命令用来删除一个标签；

git blname ‘**.txt’ 查看提交的详细信息（包括作者，改动的具体位置）
git diff 比较区中的文件区别（对象区和工作区）；
git cherry-pick 如果发现写错了分支的代码，需要转移到另外一个分支，可以使用此命令,该命令是复制操作，原有的分支代码，还会继续存在（注:sha1值会发生变化）；

git rebase 该命令用来合并分支，也称为变基操作，如果是自己独有的分支，建议使用这个命令来合并分支；（如果产生了冲突，按照正常的解决冲突的步骤，然后在执行 git rebase –continue）；

git rebase –abort 表示回到rebase之前的状态；
