http相关知识点整理

发表于 2020-02-05 更新于 2020-03-20 分类于 前端

前言

http协议是前端面试中非常重要的一环，也是频繁的考点，对此知识点掌握的详细程度，先的尤为重要了，最近复习了一些http的相关知识利用闲暇时间，总结出来，方便自己复习。当然也还有其他的协议，比如FTP协议，在这里就不在过多的赘述….

一. http 常见的状态码

200 表示成功返回数据；
204 表示成功返回，但是没有资源返回；
206 表示范围请求，服务端成功处理这段内容；

301表示重定向（永久重定向）
302表示临时重定向
303也表示重定向，但是只能用get请求；
307表示临时重定向；

400表示请求报文中存在语法错误；
401表示该请求需要通过http认证信息；
403表示请求被拒绝；
404表示资源没有找到；

500表示服务器在执行请求时内部发生错误；
502表示请求超时；

二.客户端缓存
2.1、在前端开发中,性能一直都是被大家所重视的一点,提高网页反应速度的一个方式就是使用缓存.一个优秀的缓存策略可以缩短网页请求资源的距离,减少延迟,并且由于缓存文件可以重复利用,还可以减少带宽。浏览器端的缓存可以分为强缓存和协商缓存。页面的缓存状态是由header决定的,header的参数有四种：

1.Cache-Control策略
2.Expires策略
  表示缓存过期时间,用来指定资源到期的时间,是服务器端的具体的时间。
3.Last-modified策略
  表示服务器端文件的最后修改时间,需要和cache-control共同使用。是检查服务器端资源是否更新的一种方式。
4.ETag策略
  ETag是服务端返回的一段hash字符串，浏览器会将这串字符串传回服务器,验证资源是否已经修改。

2.2 三次握手/四次挥手

  第一步：浏览器向服务器发送一个syn=1的数据包；
  第二步：服务器收到之后，返回syn和ack=1的响应包给浏览器；
  第三步：浏览器收到syn和ack=1之后，在向服务器发送一个确认，并最终建立起连接。
----------------------------------------------------------------------------------
  第一步：浏览器端发送一个关闭FIN请求到服务器端；
  第二步：服务端收到FIN后，发送一个ACK+1的序号给浏览器端；
  第三步：服务端再发送一个FIN，用来关闭服务端到浏览器端的数据传送；
  第四步：浏览器端收到FIN后，接着发送一个ACK+1信号给服务端，完成四次挥手；
